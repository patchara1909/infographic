1. บทบาทและขอบเขตงานที่รับผิดชอบ

หน้าที่หลัก: เทรน AI ตรวจจับใบหน้า, เจน QR CODE ให้อยู่ได้ 60 วินาที และพัฒนาระบบ QR Code สำหรับล็อคอิน, UI แสดงประวัติการสแกน QR Code

เป้าหมาย: สามารถนำ QR CODE มาใช้ร่วมกับ AI ตรวจจับใบหน้าได้

2. ผลการดำเนินงาน

สำหรับงานที่ได้รับมอบหมายทำได้ครบ 80%

สามารถนำ QR Code ไปสแกนร่วมกับ AI ตรวจจับใบหน้าได้แล้ว และสามารถแสดงประวัติ (วัน/เวลา) ที่มีการสแกนหรือยืนยันตัวตนขึ้นแสดงได้แล้ว

3. การใช้หลักการและเหตุผล

ทำไมถึงใช้ QR Code? เพื่อเพิ่มความปลอดภัยและความรวดเร็วในการยืนยันตัวตน

  • QR Code: ใช้ความรู้มาตรฐาน QR Code และ Dynamic QR Code เพื่อเพิ่มความปลอดภัย
  • Computer Vision: ใช้ OpenCV และ WebRTC สำหรับประมวลผลภาพและเชื่อมต่อกล้อง
  • UI Design: ออกแบบเน้นความถูกต้อง ความรวดเร็ว และความพึงพอใจ
  • Database: ใช้ MySQL สำหรับจัดเก็บข้อมูล
อ้างอิง: Denso Wave, ISO/IEC 18004, Szeliski's Computer Vision, OpenCV, WebRTC, MySQL Docs

4. การเชื่อมโยงงานกับวัตถุประสงค์ของโครงงานและภาพรวมของกลุ่ม

QR Code เปรียบเสมือนรหัสนักศึกษา ใช้ก่อนการยืนยันตัวตน และตรวจสอบวัน/เวลา

เชื่อมกับ: Database และ AI ตรวจจับใบหน้า

ผลลัพธ์ส่งต่อ: ระบบ QR Code มีฟังก์ชันครบแล้ว

5. การวิเคราะห์ แก้ปัญหา และการเรียนรู้

ปัญหาที่พบ:

  • QR Code ไม่ยืนยันตัวตนคนที่สแกน
  • เจน QR Code บางทีสแกนไม่ได้
  • ดึงข้อมูลรูปภาพมาเปรียบเทียบ
  • เรียกใช้กล้องสแกน
  • ดึงข้อมูลจาก Database

วิธีแก้: หางานวิจัยเพิ่มเติมเกี่ยวกับ Database และ QR Code

6. แผนงานระยะถัดไป

แผน: นำโปรเจครวมกับเพื่อน นำขึ้นเซิร์ฟเวอร์ใช้งานจริง

อุปสรรคคาดการณ์: การเก็บข้อมูลมาเทรนเพื่อความแม่นยำ
